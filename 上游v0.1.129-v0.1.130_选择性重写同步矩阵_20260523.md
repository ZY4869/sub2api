## 已实现

| 项目 | 状态 | 本地实现摘要 |
| --- | --- | --- |
| 兑换码批量更新 | 已实现 | 新增 `POST /api/v1/admin/redeem-codes/batch-update`，支持批量更新 `status`、`notes`、`expires_at`、`group_id`、`type`、`value`、`validity_days`；写接口沿用管理员鉴权、幂等执行和审计日志。 |
| 兑换码状态安全边界 | 已实现 | 批量 `status` 只允许 `unused`、`expired`、`disabled`；拒绝批量伪造 `used`，避免缺失 `used_by` / `used_at`。 |
| 兑换码过期参数 | 已实现 | `generate` 与 `create-and-redeem` 新增 `expires_in_days`，并与 `expires_at` 做互斥校验，冲突返回 `REDEEM_CODE_EXPIRY_CONFLICT`；批量 `expires_at: null` 可清除过期时间。 |
| 兑换码排序 | 已实现 | 列表和导出支持 `sort_by`、`sort_order`，排序字段白名单化，默认 `id desc`；前端 API 类型已对齐后端白名单中的 `group_id` / `validity_days`。 |
| 注册邮箱白名单通配 | 已实现 | 精确项 `@example.com` 只匹配根域；`@*.example.com` / `*.example.com` 只匹配子域。 |
| OIDC / OAuth verified-email 快速通道 | 已实现 | provider 邮箱已验证、无本地同邮箱冲突、站点允许注册、未强制邀请码补全且邮箱白名单通过时直接创建 / 登录，跳过二次 choice 页。 |
| OIDC / OAuth 同邮箱冲突保护 | 已实现 | 本地已有同邮箱账号时不自动接管，不凭 verified email 自动绑定或登录，继续走现有安全绑定 / 确认流程。 |
| Chat Completions `developer` role | 已实现 | OpenAI Chat Completions 转 Responses 时将 `developer` role 映射为 system 语义。 |
| 空 thinking block 兼容 | 已实现 | 转发前规范化空 `thinking` / `redacted_thinking` block；如上游仍返回 “thinking block must contain thinking” 类错误，清理后最多重试一次。 |
| Token Usage Trend Cache Hit Rate | 已实现 | 前端图表分母改为全部 prompt tokens，即 `input + cache_creation + cache_read`。 |
| 内容审核模型过滤 | 已实现 | 新增 `content_moderation_model_filter` 设置，支持 `all`、`include`、`exclude`，只决定是否调用审核，不改变路由或模型可见性。 |
| 内容审核请求内去重 | 已实现 | 同一请求内重复用户消息先去重再送审，减少 agent 工具循环导致的重复外部审核调用。 |
| API 文档同步 | 已实现 | 更新 `backend/internal/service/docs/pages/common.md`，覆盖兑换码、审核、注册 / OIDC 与兼容行为说明和示例。 |
