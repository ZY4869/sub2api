# v0.1.171 更新说明

发布日期：2026-03-31

## 重点更新

- 新增 Gemini Group 的 `AI Studio / Vertex` 混合协议兜底开关，同协议优先，额度或限流不满足时可跨协议继续选路。
- Gemini 原生路由扩展为同时支持 AI Studio `Files API / Batch API` 与 Vertex AI `batchPredictionJobs`、同步 `publishers/google/models` 入口。
- 新增上游异步资源绑定与 Google Batch 额度预留，避免多账号场景下 `file / batch / job` 串账号，并为 Batch 独立限额预检提供持久化状态。

## 管理台增强

- 后台新增全局 `Google Batch GCS Profile` 管理、激活与测试连接能力，用于 AI Studio / Vertex Batch 中转。
- AI Studio tier 升级为 `Free / Tier 1 / Tier 2 / Tier 3`，兼容旧 `aistudio_paid` 到 `Tier 1`，并新增服务端 `Gemini Rate Catalog` 供前端统一展示官方限流与文档链接。
- 修复 Gemini 渠道来源展示：`vertex_express` 与 `oauth_type=vertex_ai` 统一正确显示为 `Vertex AI`，不再误标为 `AI Studio`。

## 配额与路由修复

- 账号金额限额扩展到所有参与网关选路的账号类型，超过额度时自动跳过当前账号并继续选择其它可用账号。
- Protocol Gateway 新增 Gemini Batch 开关，开启后保持 Google 官方原生协议格式，不引入自定义 Batch DTO。

## 验证结果

- `cd frontend && npm.cmd run typecheck`
- `cd frontend && npm.cmd run test:run`
- `cd frontend && npm.cmd run build`
