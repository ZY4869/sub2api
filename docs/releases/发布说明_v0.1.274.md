# v0.1.274 更新说明

发布日期：2026-04-23

## 重点更新

- 修复 OpenAI native Images 链路在 `stream=true` 时的兼容性：`/v1/images/generations`、`/v1/images/edits` 支持原生 SSE 透传，不再因按 JSON 解析导致失败。
- 补齐 `/v1/responses` 生图 tool（`image_generation`）在 `stream=true` 下的“实际出图数”统计，完善图片计费调度命中率（按出图数 + 尺寸档）。
- 管理后台账号测试升级为通用生图测试：OpenAI `gpt-image-2` 可输入 prompt 并预览图片（SSE `type:"image"` 事件）。

## 后端能力补充

- native Images `stream=true`：新增 SSE proxy 分支并按 `*.completed` 事件统计出图数，写入 usage 记录所需字段。
- Responses tool `stream=true`：从 `response.image_generation_call.partial_image.output_index` 去重统计出图数，结束前写入 `image_output_count` request metadata，保持计费 resolver 链路不变。
- 管理后台测试：OpenAI GPT image 模型不再被“已知模型回退”覆盖，按 images 协议发起真实请求并输出 `image` 事件。

## 前端联动

- 账号测试弹窗从 Gemini 专用生图测试泛化为通用生图测试：图片模型（OpenAI/Gemini）均展示 prompt 输入与图片预览。
- 更新中英文 i18n 文案 key（`imageTest*`）并补齐 Vitest 覆盖 OpenAI `gpt-image-2` 流程。
