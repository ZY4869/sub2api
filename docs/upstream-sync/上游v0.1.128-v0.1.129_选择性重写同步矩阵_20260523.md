## 融合实现说明

| 项目 | 决策 |
| --- | --- |
| 定价同步 | 采用本地 billing pricing snapshot 架构实现。新增同步入口只更新官方层/快照并保留 sale 层，避免覆盖本地商业售价。 |
| Codex OAuth UA | 与本地 `ForceCodexCLI` 和账号自定义 UA 兼容；策略只在 OpenAI OAuth/Codex 账号请求中生效。 |
| 图片协议 | 保持本地 `image_protocol_mode` 三层优先级，不改为按模型名猜测链路。 |
| 模型目录 | 只补本地 registry/映射，不允许 probe snapshot 或 known models 扩展可见模型集。 |
| 邮件模板 | 采用本地内置模板作为兜底源，自定义模板只保存管理员改动；模板渲染失败只 fallback 一次，避免递归或静默吞错。 |
| 通知偏好 | 先开放支付成功、余额提醒、订阅到期三类用户偏好；验证码和密码重置保持安全邮件，不受用户退订影响。 |
| Ops retry | 不删除现有 trace/preview 能力，只在重试入口做 body 安全闸门；被截断、脱敏或含凭据的历史请求明确拒绝重试。 |
