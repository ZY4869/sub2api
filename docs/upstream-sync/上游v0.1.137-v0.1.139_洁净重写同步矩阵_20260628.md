- 功能边界：同步 v0.1.137-v0.1.139 的安全、稳定性、兼容性和必要产品能力；不新增独立 API 文档中心，不添加 `/api-docs/*` 或 `/admin/api-docs/*`。
- 入口与触点：Admin 设置/账号/用量/支付/渠道监控、OAuth 注册登录、OpenAI/Grok/Claude/Gemini/Vertex 网关、scheduler outbox、计费钱包、`sub2api-admin` CLI、Docker Compose 部署配置。
- API 契约：保留本地统一错误结构；`model_not_found` 维持 404；设置、支付、渠道监控、账号调度字段只做向后兼容扩展。
- 数据层：优先复用本地迁移编号；已存在 `146_channel_monitor_jitter.sql` 与 `147_scheduler_outbox_dedup_cleanup.sql`，不使用上游迁移编号。
- 模型调用：继续走本地 gateway/service/client 封装；读路径不得触发同步下游模型探测；模型可见性遵守 `model_scope_v2.entries[]`。
- 日志与审计：关键失败记录 requestId/correlationId、账号 ID、错误分类和耗时；不记录 token、PAT、refresh token、PII 明文或上游许可文本。
- 权限/配额/限流：`SUB2API_ADMIN_TOKEN` 优先于 `SUB2API_JWT`；`codex_cli_only`、OAuth 后缀白名单、promo/affiliate、支付扣款、余额 hold 与重置限额均走既有鉴权和幂等链路。
- 指标与告警：复用现有 ops/gateway/usage/billing 指标；本轮只补关键锚点守卫，不新增孤立指标名。
- 发布与兼容：根 `LICENSE`、README 许可口径、frontend/backend 版本线和商业发布约束不变。
