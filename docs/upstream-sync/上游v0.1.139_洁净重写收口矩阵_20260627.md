## Integration Map

- 功能边界：收口 CLI 认证 fallback、OAuth 邮箱身份创建错误遮蔽修复，并确认 OpenAI/Codex/Grok、支付计费、调度、Admin UX 已有能力未回退；不新增强制开关。
- 入口与触点：`skills/sub2api-admin`、OAuth/SSO 登录注册、OpenAI 兼容网关、Grok 专属路由、支付 provider、scheduler outbox、Admin 账号/用量/Ops 页面。
- API 契约：保留本地统一错误体；`model_not_found` 继续返回 404；OAuth 创建冲突恢复失败仍对外映射为 `SERVICE_UNAVAILABLE`，错误链保留原始 email conflict 语义。
- 数据层：本轮不新增迁移；确认已有 channel monitor jitter 与 scheduler outbox dedup/cleanup 迁移按本地编号存在。
- 模型调用：OpenAI/Grok/Codex 相关行为继续走本地 gateway/service/client 封装和既有脱敏观测字段。
- 日志与审计：CLI 不输出 token；OAuth 冲突恢复失败记录创建错误与查询错误，不记录 token、refresh token、PAT 或 PII 扩展字段。
- 权限/配额/限流：`SUB2API_ADMIN_TOKEN` 优先于 `SUB2API_JWT`；admin 操作继续使用现有鉴权；`codex_cli_only` 继续在相关 OpenAI 入口统一约束。
- 指标与告警：继续复用本地 ops、usage、gateway 错误分类、调度 outbox 观测链路；本轮不新增指标名。
- 测试与回归：补 CLI fallback/优先级测试、OAuth 冲突失败测试；对已存在网关/Admin/调度能力保留现有测试集作为回归锚点。
- 发布与兼容：新增行为向后兼容；无 license、README、public model example template 破坏性变更。
