# 上游 v0.1.135 洁净重写同步计划与矩阵（2026-06-08）

## 范围与边界

- 目标 upstream：`Wei-Shaw/sub2api`，参考 release `v0.1.135` 与 `v0.1.134...v0.1.135` 的行为变化、缺陷线索和测试意图。
- 本地功能优先；只按本地架构重写吸收安全、网关、调度、用量、代理韧性和 admin 运维能力。
- 不执行 `pull`、merge、cherry-pick，不复制 upstream LGPL 源码、脚本、测试或文案。
- 跳过 upstream `LICENSE`、发布元数据、README 安装文案、版本号回退、LGPL 原始脚本和独立 API 文档中心。
- 本地根 `LICENSE` 保持 MIT；不新增 `/api-docs/*` 或 `/admin/api-docs/*`。

## Integration Map

- 功能边界：吸收代理有效期/fallback、OpenAI transport failover、API Key 独占分组、previous response 粘性校验、5h 窗口显示、缓存 token 聚合和 admin Skill；不引入 upstream 许可文本或发布资产。
- 入口与触点：`/v1/*` OpenAI 网关、API Key 鉴权、admin accounts/proxies/ops、后台周期任务、代理导入导出、代理管理 UI、usage UI、`skills/sub2api-admin`。
- API 契约：扩展 proxy/account DTO；新增账号回切原代理 admin 动作；usage 聚合补充 cache creation/read/total/hit rate；错误仍使用本地统一响应模型。
- 数据层：新增本地迁移 `138_proxy_expiry_fallback.sql`；`proxies` 增加有效期、提醒天数、fallback 代理；`accounts` 记录被 fallback 替换前的原代理。
- 模型调用：未新增模型调用入口；OpenAI 请求继续走本地 gateway、scheduler、rate-limit、temp-unschedule 封装。
- 日志与审计：代理到期扫描、fallback 切换、账号回切、OpenAI transport failover、独占分组拦截均走脱敏日志和既有 ops 上下文。
- 权限/配额/限流：admin 端点继续走 adminAuth；API Key 分组同时校验 key 绑定和用户 allowed groups；后台扫描幂等；Skill 只使用 admin HTTP token。
- 指标与告警：ops alert evaluator 增加 `account_temp_unscheduled_count`，transport failover 触发临时不可调度后进入现有可观测链路。
- 测试与回归：覆盖迁移、代理闭环、failover、Content-Type、独占分组、previous response 粘性、5h `resets_at`、缓存统计、前端代理 UI 和 Skill dry-run。
- 发布与兼容：新增字段均向后兼容；旧导入 payload 缺字段按空值处理；保留 MIT 许可证。
