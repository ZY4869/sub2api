# v0.1.275 更新说明

发布日期：2026-04-23

## 重点更新

- 修复 OpenAI `gpt-image-2` 在 ChatGPT/Codex OAuth 形态下无法直接走原生 Images 的问题：当图片协议为 `compat` 时，`/v1/images/generations`、`/v1/images/edits` 会桥接到上游 `POST /v1/responses`，并把顶层 `model` 固定路由为 `gpt-5.4-mini`，tool 内仍保持 `model=gpt-image-2`，确保兼容生图可用且计费模型不漂移。
- 增强 `POST /v1/responses`（仅 JSON）：支持 `model=gpt-image-2` 作为“自动生图简写”，无需 `$imagegen ` 前缀即可注入 `image_generation` tool，并允许 `image_generation` / `reference_images` / `mask` 在无前缀时生效；当 `tool_choice` 与生图意图冲突时返回 `imagegen_compat_tool_choice_conflict`。
- 文档同步：补齐 `openai-native` 与 `common` 页面关于 `model=gpt-image-2` 简写与内部 host model 路由规则的说明。
